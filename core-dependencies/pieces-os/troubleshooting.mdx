### Database & Log Paths

You can find your database and support logs at the paths below for your specific platform:

- **macOS**—`~/Library/com.pieces.os/`
- **Windows**—`C:\Users\<username>\Documents\com.pieces.os\`
- **Linux**—`~/.local/share/com.pieces.os/`

> Don’t forget to replace `<username>` with your actual system username.

### Logs, Backups & Cloud Functionality

For more details on logs, backups, and cloud-only features, see the [On-Device Storage documentation](https://docs.pieces.app/products/core-dependencies/on-device-storage).

- **Sharing Logs**—Zip the latest 2–3 files from `…/production/support/logs/` and attach them to [your issue or DM](https://docs.pieces.app/products/support#open-a-github-issue).
- **Backup & Reset**—Copy the entire `com.pieces.os` folder to back up your database; or zip and rename it (anything other than `production`) to reset your database on next launch.
- **Cloud Functionality**—Nothing moves off-device unless you explicitly enable Personal Cloud or call a third-party model (e.g. OpenAI).

### Checking System Specifications

Navigate between the tabs below to verify your hardware meets the requirements for running PiecesOS and any local AI models.

<Tabs>
  <Tab title="macOS">
    **macOS | Checking System Specifications**

    Apple Silicon (ARM) and Intel devices use different CPU architectures. Installing the wrong build will prevent PiecesOS from running.

    1. Click the **Apple () icon** in the top-left corner.
    2. Select **About This Mac**.
    3. In the **Overview** section, check the first line:
       - **Apple Silicon / ARM:** e.g. `Apple M3`
       - **Intel:** e.g. `2.6 GHz Intel Core i7`

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/macos/macos_checking_about_mac.gif)
  </Tab>
  <Tab title="Windows">
    **Windows | Checking System Specifications**

    1. Press the **Windows** key (or click the Windows icon).
    2. Type `run` and press **Enter**.
    3. Enter `dxdiag` and click **OK**.
    4. In the **System** tab, view your processor, core count, and RAM.
    5. In the **Display** tab, view GPU model and VRAM.

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/windows_checking_hardware_specs.gif)
  </Tab>
  <Tab title="Linux">
    **Linux | Checking System Specifications**

    - **CPU Info:**\
      lscpu

      ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/linux/lscpu_UBUNTU.png)
    - **GPU Info:**\
      lspci | grep -i vga

      ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/linux/lscpi_UBUNTU.png)

### Versions & Updates

Out-of-date plugins or PiecesOS versions are a common source of issues. Use the tabs below to learn how to check for and install the latest updates.

<Tabs>
  <Tab title="macOS">
    **macOS | Updating PiecesOS**

    1. Quit and restart PiecesOS.
    2. Look for the **Pieces** icon in your menu bar.
    3. If an update is available, follow the prompt to download and install.

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/macos/macos_checking_piecesos_for_updates.gif)
  </Tab>
  <Tab title="Windows">
    **Windows | Updating PiecesOS**

    1. Click the **P** icon in your taskbar.
    2. If an update is listed, follow the on-screen instructions to install.

    _`Note: If installed via .appinstaller, fully quit both the Desktop App and PiecesOS before rerunning the installer.`_

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/windows_checking_pieces_os_for_updates.gif)
  </Tab>
  <Tab title="Linux">
    **Linux | Updating PiecesOS**

    PiecesOS is distributed via snaps on Ubuntu-based systems:

    snap info pieces-os\
    sudo snap refresh

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/linux/snap_refresh_POS.gif)
  </Tab>
</Tabs>

### Checking OS Version

An outdated operating system can block installation or prevent PiecesOS from launching. Use the tabs below to verify your OS meets minimum requirements.

<Tabs>
  <Tab title="macOS">
    **macOS | Minimum Version**

    Pieces Suite requires **macOS Ventura** or later.

    1. Click **Apple ()** → **About This Mac**.
    2. Look for the **macOS** line (e.g., `Sequoia 15.1.1`).

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/macos/macos_checking_about_mac.gif)
  </Tab>
  <Tab title="Windows">
    **Windows | Updating OS**

    1. Open **Start** → **Settings** → **Windows Update**.
    2. Install any pending updates and reboot.

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/checking_windows_ver.gif)
  </Tab>
  <Tab title="Linux">
    **Linux | Minimum Version**

    PiecesOS requires **Ubuntu 22.04** or newer.

    1. Open **Settings** → **System** → **About**.
    2. Check the **Operating System** label.

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/linux/settings_about_system.gif)
  </Tab>
</Tabs>

### Issues Launching PiecesOS (Windows)

Some Windows users with **Controlled Folder Access (CFA)** enabled may see an error when PiecesOS tries to write data.

To resolve:

1. Open **Windows Security** → **Virus & threat protection** → **Manage ransomware protection**.
2. Under **Controlled folder access**, click **Allow an app through Controlled folder access**.
3. Add the PiecesOS executable (`com.pieces.os.exe`) to the allowlist.

<Warning>
  PiecesOS fails to launch when CFA is enabled because the executable path for the application writes data to the Documents folder.

  CFA disables and blocks any request to modify files (in this case, writing & saving data), so PiecesOS cannot launch itself.
</Warning>

You can also disable CFA entirely if it’s not required in your environment. To manage specific app permissions, visit the [Long-Term Memory Access Control panel](https://docs.pieces.app/products/core-dependencies/pieces-os/quick-menu#long-term-memory-access-control).
