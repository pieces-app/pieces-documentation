## Installation & Updating Fixes

PiecesOS and the Pieces Desktop App can be downloaded in several ways, and the update process varies based on the method you used, like whether you installed them via WinGet or `.exe` files.

<Tabs>
  <Tab title="On-Device Storage">
    All your Pieces data, including snippets, tags, embeddings, LTM-2 memory, settings, logs, and Pieces Copilot history, is securely stored in the `com.pieces.os` and `com.pieces.pfd` folders on your device.

    This gives you _full control_ over backups, sharing, and privacy.

    ![](https://storage.googleapis.com/hashnode_product_documentation_assets/privacy_and_security/soc_secure_private.png)
  </Tab>
</Tabs>

---

### Database & Log Paths

You can find your database and support logs at the paths below for your specific platform:

- **macOS**—`~/Library/com.pieces.os/`
- **Windows**—`C:\Users\<username>\Documents\com.pieces.os\`
- **Linux**—`~/.local/share/com.pieces.os/`

Replace `<username>` with your actual Windows user name.

### Logs, Backups & Cloud Functionality

For more details on logs, backups, and cloud-only features, see our [On-Device Storage documentation](https://docs.pieces.app/products/core-dependencies/on-device-storage). You’ll learn how to:

- Zip and share logs from `…/production/support/logs/` when opening an issue.
- Back up or reset your database by copying or renaming the `com.pieces.os` folder.
- Enable or disable cloud functionality.

## Versions & Updates

Keeping both PiecesOS and the Desktop App up-to-date often resolves common issues.

### Updating PiecesOS

1. Click the **Pieces** icon (`P`) in your taskbar.
2. View your update status and follow the prompt to install any updates.

![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/windows_checking_pieces_os_for_updates.gif)

### Updating the Desktop App

1. Open the Pieces Desktop App.
2. Click the **User Icon** next to the Search Bar.
3. Select **Check for Desktop App Updates** (or **Check for PiecesOS Updates**).

![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/windows_check_pfd_for_updates.gif)

## Issues Launching PiecesOS

If Controlled Folder Access (CFA) is enabled, PiecesOS may fail to write data and launch. To fix:

1. Add the PiecesOS executable path to your CFA allowlist.
2. Or disable CFA if not required.

<Warning>
  CFA blocks write access to Documents, preventing PiecesOS from launching.
</Warning>

## Common Installation Issues

### Checking for Windows Updates

1. Open **Settings** → **Windows Update**.
2. Install any pending updates and reboot.

### Updating Microsoft Store & App Installer

1. Open **Microsoft Store** → **Library**.
2. Update the **Store** and **App Installer**.
3. Retry the Pieces installation.

## Checking Hardware Specifications

Run **Windows \+ R**, type `dxdiag`, and press Enter to view CPU, RAM, GPU, and VRAM details—essential for troubleshooting local LLM performance.

![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/windows_checking_hardware_specs.gif)

[Minimum hardware recommendations](https://docs.pieces.app/products/meet-pieces/troubleshooting/cross-platform#hardware-recommendations)

### Checking Windows Version

Press **Windows \+ R**, type `winver`, and press Enter. The **About Windows** window shows your OS version.

![](https://storage.googleapis.com/hashnode_product_documentation_assets/meet_pieces_assets/meet_pieces/troubleshooting/windows/checking_windows_ver.gif)

<Tip>
  If `winver` doesn’t open, you’re likely on an unsupported Windows version.
</Tip>
